代码是人类创造的高质量语言之一，通过高度的抽象来代替形式多样的自然语言，最终转换为具体程序来代替人类完成任务，其具有精确性、逻辑性和可执行性等优点。所以代码能力也成为大语言模型（LLMs）的核心能力，我们期待 LLMs 可以帮助人类进行辅助编程、漏洞修复、甚至是全自动代码生成等工作。

如何通过高质量的数据微调提升基础语言模型的代码能力仍然是一个开放且具有挑战的问题，Qwen AI挑战赛由阿里云和NVIDIA主办，天池平台和魔搭联合承办，是聚焦于通义千问大模型微调训练的竞赛，其主要目标是通过高质量的数据探索和拓展开源模型 Qwen 1.8B 及 Qwen 72B 的代码能力上限。

PHI-1在HumanEval上能够达到51%的准确率，其核心方法介绍如下:

数据集：

  1. Filtered Code-Language Dataset（过滤后的代码-语言数据集）：
    - 数据类型：这个数据集是从The Stack和StackOverflow中提取的，包含了Python代码和相关的自然语言注释。
    - 规模：这个数据集包含了大约6亿（6B）（tokens）。

  2. Synthetic Textbook Dataset（合成教材数据集）：
    - 数据类型：这个数据集由GPT-3.5合成生成，包含了Python教材的自然语言文本和相关的代码片段。
    - 规模：这个数据集包含了少于10亿（1B）个token。

  3. CodeExercises Dataset（编程练习数据集）：

- 数据类型：这个数据集包含了Python编程练习和相应的解决方案的docstring。

- 规模：这个数据集包含了大约1.8亿（180M）个token。

训练方法：

  1. 预训练：phi-1-base在CodeTextbook数据集(包含Filtered Code-Language Dataset和Synthetic Textbook Dataset)上训练了8轮约50B Token

  2. 微调：在phi-1-base 通过CodeExercises数据集进行微调。

作者在训练过程中使用了这三个主要数据集的组合，以便为phi-1模型提供不同类型的训练信号和任务。首先，他们使用了过滤后的代码-语言数据集（Filtered Code-Language Dataset）来进行模型的预训练，然后使用合成教材数据集（Synthetic Textbook Dataset）来进一步提高模型的性能。最后，通过使用编程练习数据集（CodeExercises Dataset）进行微调，他们使模型能够更好地生成与编程任务相关的代码。

这些不同类型的数据集在训练过程中的使用，有助于模型学习不同层次和类型的知识，从而提高了模型的性能和通用性。此外，这种训练方法还强调了数据质量的重要性，特别是高质量教材数据在提高模型性能方面的作用。

高质量的教材数据与传统的代码数据有一些显著的区别：

1. 清晰和自包含性（Clarity and Self-Containment）：高质量的教材数据具有清晰的表达和自包含性，使得其中的内容可以独立理解，而不需要额外的上下文。相比之下，传统的代码数据可能包含对其他模块或文件的依赖，这会使得理解起来变得困难。

2. 教学性（Instructiveness）：高质量的教材数据应当具有教学性质，能够有效地教授基本编码概念。然而，传统的代码数据中可能包含了大量的非教学性内容，例如一些极为简单或者样板式的代码，这些代码不涉及任何有意义的计算。

3. 算法逻辑（Algorithmic Logic）：高质量的教材数据应当涵盖算法逻辑，而不仅仅是提供一些常量定义、参数设置等简单的代码片段。相比之下，传统的代码数据中可能会存在算法逻辑被隐藏在复杂或者缺乏文档的函数内部的情况。

4. 平衡性（Balance）：高质量的教材数据应当涵盖广泛的编码概念和技能，而不应该偏向特定主题或用例。传统的代码数据可能会存在某些主题或用例的偏向，导致数据集中编码概念和技能的分布不均衡。

作者认为高质量的教材数据对模型训练至关重要的原因在于：

1. 提升信号质量：高质量的教材数据可以提供更为清晰和有效的信号，帮助模型将自然语言映射到代码，从而提高模型的性能。

2. 减少噪声和不完整性：传统的代码数据可能存在大量的噪声、模糊和不完整的信息，这会干扰模型的学习过程。高质量的教材数据能够减少这些问题，使模型得到更为准确和完整的训练。

3. 提升模型通用性：通过使用高质量的教材数据，模型可以学习到更广泛、更多样化的编码概念和技能，从而提升模型在不同任务上的通用性和鲁棒性。

综上所述，高质量的教材数据能够为模型提供更为有效、清晰、有教学性的训练信号，使其在编码生成任务中取得更好的性能。

参考论文2:

https://arxiv.org/pdf/2312.02120.pdf

MagicoderS-CL-7B 通过在高质量数据集OSS-Instruct数据集上SFT，显著提高了模型在代码生成任务上的表现。

OSS-Instruct数据集生成的Prompt如下：

OSS-Instruct核心思想：将开源代码数据作为种子代码片段，生成高质量的代码生成任务及解决方案。

从80K个代码文档中收集了80K个初始种子片段，其中包括40K的Python，以及C++，Java，TypeScript，Shell，C#，Rust，PHP，和Swift各5K。然后，每个收集到的种子代码片段被应用到图2所示的提示模板，一个教师模型将其作为输入，并输出一个代码问题及其解决方案。

数据生成示例：

模型训练：

  1. 第一阶段75K OSS-Instruct数据集训练2轮，学习率5e-5

  2. 第二阶段110K evol-codealpaca-v1数据集，相同参数配置

总的来说，OSS-INSTRUCT是一种使用大型语言模型生成来自开源代码片段的低偏差和高质量编码挑战的新型数据生成方法。这种方法使得Magicoder显著提升了基础的大型语言模型（LLM）。尽管参数少于70亿，它能够胜过所有参数小于或等于160亿的评估LLM，包括150亿参数的WizardCoder。将OSS-INSTRUCT与Evol-Instruct相结合构建增强的MagicoderS模型。它们通过与领先模型如ChatGPT在HumanEval基准测试中的比较，取得了显著成果。

解决方案

  • OSS-Instruct数据集表现由于比赛时间，暂未混合训练进行对比～

  • 训练方式采用全量参数微调，计算资源充足情况下直接SFT效果会优于LoRA训练。

        原因可见论文：https://arxiv.org/pdf/2304.08109.pdf

  • 训练数据泄漏检测：离线针对训练数据做了泄漏检测：输入输出的相似去重；同名函数的去重

  • 初赛模型已开源：https://modelscope.cn/models/JMXGODLZ/CodeQWen_1.8B/summary

复赛方案

赛题分析

复赛数据格式:

根据赛题和主办方群里答疑，可以得出赛题的关键点如下：

  1. 复赛代码语言只包含Python语言

  2. 复赛题目为LeetCode数据

  3. 复赛程序需要直接运行无报错，包的引用、空格对齐、运行结果、时间复杂度均需要符合题目要求

  4. 通过测算，72B的QWen模型在进行Lora训练时，针对比赛所提供资源，使用1万条样本约需4.5小时。因此，我们优先考虑数据的相关性与高质量，而非单纯追求庞大的数据量，以确保整体训练的高效性与有效性。

解决方案

  1. 收集LeetCode相关的Python语言开源数据

  2. 收集LeetCode相关解题思路，用于代码解释

难点：

本地执行复赛验证集推理代码时，出现以下问题：

  • 执行代码缺失引用包，比如math、heapq这些外部库

  • 生成内容涵盖错误的注释以及测试用例，导致无法正常运行通过测试

  • 同名函数导致模型生成解决方案未遵循题目要求、时间复杂度要求

为了解决前两个问题，我通过清理训练数据、整理常用的外部库、删除未被引用的库以及生成测试数据。

至于最后一个也是最具挑战性的问题，由于时间原因，尚未进行尝试。预计的解决方案是对函数名称进行重构。
